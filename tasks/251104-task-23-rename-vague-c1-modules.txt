TASK-23: Rename Vague C1 Module Names
======================================

Date: 2025-11-04
Status: Active  
Priority: HIGH (do before continuing extraction)

## Problem

Several c1 modules have vague names that don't clearly indicate what they contain:

1. **c1_validation_enums** - Vague. Validation of what? Enums for what validation?
2. **c1_validation_models** - Vague. Models for what kind of validation?
3. **c1_validation_utils** - Vague. Utilities for what validation?

General module names are problematic because:
- Not immediately clear what the module contains
- Could mean many different things (data validation? task validation? result validation?)
- Makes imports ambiguous:  - what is being validated?

## Principle: Module Names Must Be Specific

**GOOD module names:**
- c1_task_validation_enums (enums for task validation checks)
- c1_result_validation_models (models for result validation)
- c1_data_validation_utils (utilities for data validation)

**BAD module names:**
- c1_validation_* (validation of WHAT?)
- c1_config_* (config for WHAT?)
- c1_utils_* (utils for WHAT?)

## Analysis: What Do These Validation Modules Actually Do?

### c1_validation_enums/

**Current name**: c1_validation_enums
**Contents**: ValidationCheckType enum
**Usage**: Used by result validation system (checking if task results meet criteria)
**Better name**: c1_result_validation_enums

**Reasoning**: These enums define types of validation checks for task RESULTS, not general validation.

### c1_validation_models/ (if exists)

**Current name**: c1_validation_models
**Contents**: Unknown - need to check if this exists
**Proposed action**: Investigate, rename based on actual purpose

### c1_validation_utils/  

**Planned name**: c1_validation_utils
**Contents**: validation_helpers.py, result_validation_helpers.py
**Better name**: c1_result_validation_utils

**Reasoning**: These utilities validate task RESULTS, not general validation.

## Renaming Procedure

For each module being renamed:

1. Create new directory with specific name
2. Copy all files from old directory
3. Update __init__.py in new directory
4. Find all imports of old module name
5. Update all imports to new module name
6. Run tests to verify
7. If tests pass: Delete old directory
8. Commit with message: Rename {old} → {new} for clarity
